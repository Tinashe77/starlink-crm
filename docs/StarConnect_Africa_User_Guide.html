Getting Started

Login, Roles, and Daily Navigation

Sign In
  1. Open the portal login page.
  2. Enter your email address and password.
  3. Select Sign in.
  4. After authentication, you will land on the main dashboard.
Navigation
  1. Use the left sidebar to move between modules.
  2. The top header shows the current page and your active role.
  3. Your profile menu allows password change and logout.
Role Visibility
Role What They Normally Manage
Admin Users, packages, settings, contracts, payment plans, collections, and oversight
Agent Applications, customer onboarding, contract generation, and assisted payment actions
Collections Officer Installments, reminders, receipt lookups, voids, arrears, and settlements
Customer View contracts, sign agreements, and monitor statements or payment progress
Security Notes
  • Inactive users are blocked from active use of the system.
  • Only authorized roles can access finance and admin functions.
  • Customers can only view or sign records that belong to them.
  • Always log out after using shared or public computers.

Module Guide

Applications, Packages, and Contracts

1. Customer Applications
1
Open Applications and create a new customer application.
2
Capture customer details, contact details, installation site, payment details, package choice, and declaration details.
3
Submit the application, then move its status through review until it is approved or rejected.
4
Approved applications are made available for contract creation.
2. Packages and Settings
  • Packages define financial terms used by contracts.
  • Inactive packages are excluded from new applications.
  • System settings control:
    • Late fee amount
    • Grace period days
    • Early settlement discount percentage
    • EcoCash merchant code
3. Contracts
  • Create contracts from approved applications.
  • The system pulls pricing from the selected package.
  • Contract numbers are generated automatically.
  • Deposit status can be updated from the contract record.
  • Customers can sign contracts online and download a copy.
Important: A contract must be created from an approved application, and the deposit should be marked as paid before generating a payment plan.

Finance Workflow

Payment Plans, Installments, and Statements

4. Generate a Payment Plan
  1. Open Payment Plans.
  2. Locate a contract with deposit paid and no schedule.
  3. Select Generate Plan.
  4. Choose the first due date.
  5. The system creates the linked installment schedule.
5. Record a Payment
  1. Select Record Payment from the contract row or unpaid installment row.
  2. Enter the amount received, payment date, method, and optional EcoCash reference.
  3. Save the transaction.
  4. The installment is updated automatically.
  5. If all installments are paid, the contract is marked Completed.
6. View the Statement
  1. Select View Statement for the contract.
  2. The system generates the 8-week payment tracking card.
  3. The statement shows due dates, payment dates, receipt numbers, and completion status.
  4. Use browser print to save or print a hard copy.
7. Settle Balance
  • If a customer wants to clear the remaining balance at once, select Settle Balance.
  • The system settles all remaining unpaid installments in one action.
  • If enabled, the early settlement discount is applied automatically.
  • The contract is then recalculated and normally moves to Completed.

Collections

Receipts, Reminders, and Arrears Management

8. Collections Dashboard
  • Shows due today, overdue amounts, default-risk items, and recent payments.
  • Highlights the collections queue for follow-up.
  • Provides quick links to statements for at-risk customers.
9. Automated Daily Processing
  • The system checks installment status daily.
  • Pending installments can move to Late or Defaulted based on settings.
  • Reminder emails are sent as Due Soon, Due Today, or Overdue.
10. Receipt Lookup and Voiding
  1. Open Collections.
  2. Use Receipt Lookup to search by receipt number.
  3. Review the linked customer, contract, installment, and payment method.
  4. If necessary, authorized roles can void the receipt.
  5. Voiding reverses the payment from the linked installment and recalculates the contract.
Collections best practice: Use the queue daily, confirm customer contact details are correct, and follow up on overdue accounts before they move into default status.

Common Scenarios

Recommended Step-by-Step Workflows

New Customer Onboarding
  1. Create the application.
  2. Review and approve it.
  3. Generate the contract.
  4. Send the customer to sign online.
  5. Mark deposit paid.
  6. Generate the payment plan.
Weekly Collections Flow
  1. Check the Collections dashboard.
  2. Review due today and overdue items.
  3. Record incoming payments.
  4. Send reminders where needed.
  5. Use receipt lookup for reconciliations.
Customer Pays Off Early
  1. Open the contract in Payment Plans.
  2. Select Settle Balance.
  3. Choose whether to apply the early settlement discount.
  4. Confirm the settlement details.
  5. The remaining schedule is closed automatically.
Password or Access Issue
  1. Use Forgot Password on the login page.
  2. The system sends a reset email.
  3. If the account is inactive, an administrator must reactivate it.
Support Checklist
  • Confirm the user role first.
  • Check that the correct contract or application status is set.
  • Verify deposit status before expecting payment-plan actions.
  • Confirm package status if a package is not selectable.
  • Use receipt lookup if payment data appears inconsistent.
